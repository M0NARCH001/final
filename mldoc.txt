Nutrimate ML Pipeline Documentation
1. Overview
The Nutrimate ML system is a Hybrid Recommendation Engine that combines expert-defined nutritional rules with a machine learning model to personalize food suggestions.

Rule-Based System (70% weight): Ensures recommendations meet nutritional targets (e.g., "High Protein", "Low Sodium") and strictly enforces health constraints (e.g., Diabetes, Hypertension).
ML Model (30% weight): Learn's user preferences over time by analyzing which recommended foods are actually "added" by users.

3. Model Details
Algorithm
Type: Random Forest Classifier (sklearn.ensemble.RandomForestClassifier)
Configuration:
n_estimators: 100 trees
max_depth: 8 (prevents overfitting)
class_weight: "balanced" (handles class imbalance since "adds" are rarer than "views")
Features (Input Vector)
The model takes 10 numeric features representing the context of the recommendation:

Deficits (Normalized):
User's remaining Calorie need
Protein deficit
Fat deficit
Carbohydrate deficit
Food Item Properties (Normalized):
Calories
Protein content
Fat content
Carbohydrate content
Context:
rank: Only the rank assigned by the rule-based system
rule_score: The raw score from the rule-based system
Output
Target: Probability that the user will 
add
 the food item.
Hybrid Score Formula: $$ Score = 0.7 \times RuleScore + 0.3 \times (MLProbability \times 100) $$
4. Pipeline Workflow
A. Data Collection
Impression Logging: Every time recommendations are displayed, they are logged to the RecommendationImpression table via /impressions/batch.
Feedback Loop: When a user adds a food, the corresponding impression is updated (added=True).
B. Training Triggers
Training runs automatically in the background when all of these conditions are met:

Data Volume: At least 80 total impressions recorded.
Positive Samples: At least 15 successful "adds".
Time: At least 6 hours since the last training session.
C. Training Process (
ml/trainer.py
)
Fetches all impressions from the database.
Joins with FoodItem data to reconstruct features.
Splits data (80% train / 20% test).
Trains the Random Forest model.
Saves rf_recommender.joblib and scaler.joblib to the ml_artifacts/ directory.
Auto-Version Control: Pushes the new model and performance metrics to the GitHub repository automatically.
D. Inference (
ml/predictor.py
)
On API startup, the model is loaded into memory.
For each candidate food item, features are extracted.
The model predicts the probability of "acceptance".
This probability adjusts the final score to boost items the user is likely to prefer.
